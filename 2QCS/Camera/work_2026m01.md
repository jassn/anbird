# 2026-Jan-0114
Michael 測試 CAN bus.     
問題: can0 and can1 是否相反?
結果: spi 可以用 aliases 指定, 但 can 似乎不行.

### Swap spi0 and spi1
用 adb shell 進去看, 原本系統預設如下:
```
cd /sys/class/spi_master
ls -al
spi0 -> ../../devices/platform/soc/8c0000.qcom,qupv3_2_geni_se/884000.spi/spi_master/spi0
spi1 -> ../../devices/platform/soc/8c0000.qcom,qupv3_2_geni_se/88c000.spi/spi_master/spi1

```

修改 /Vendor/kernel_platform/qcom/proprietary/devicetree/qcom/kalama.dtsi, 加上 spi0 這一行:
```
 aliases: aliases {
        serial0 = &qupv3_se7_2uart;
        hsuart0 = &qupv3_se14_4uart;
        hsuart1 = &qupv3_se4_4uart;
        spi0 = &qupv3_se11_spi;
        ...
 };
```

再測試, 就可以把 spi0 指定給 0x884000.spi, 跟線路圖一致.
```
spi0 -> ../../devices/platform/soc/8c0000.qcom,qupv3_2_geni_se/88c000.spi/spi_master/spi0
spi1 -> ../../devices/platform/soc/8c0000.qcom,qupv3_2_geni_se/884000.spi/spi_master/spi1
```



### GPIO_62 conflict
Problem from adb logcat:
```
 E kalama-pinctrl f000000.pinctrl: pin GPIO_62 already requested by 884000.spi; cannot claim for ac15000.qcom,cci0:qcom,cam-sensor10
 E kalama-pinctrl f000000.pinctrl: pin-62 (ac15000.qcom,cci0:qcom,cam-sensor10) status -22
 E kalama-pinctrl f000000.pinctrl: could not request pin 62 (GPIO_62) from group gpio62  on device f000000.pinctrl
```

原因: GPIO_62 (or 884000.spi) 已被 CAN 所使用. (see DTS: qupv3_se9_spi)

解決: 修改 Vendor/vendor/qcom/proprietary/camera-devicetree/kalama-camera-sensor-rb5gen2-dvt.dtsi. 上述錯誤訊息就不會再出現！    
status = "disabled"
```
qcom,cam-sensor10 {
        cell-index = <10>;
        ......
        gpios = <&tlmm 107 0>,
            <&tlmm 72 0>,
            <&tlmm 62 0> ;
        status = "disabled";
}
```
